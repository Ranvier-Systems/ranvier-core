--------------------------------------------------------------------------------
 ACCESSING RANVIER
--------------------------------------------------------------------------------

{{- if .Values.ingress.enabled }}
Ranvier is accessible via Ingress at:
{{- range $host := .Values.ingress.hosts }}
  {{- range .paths }}
  http{{ if $.Values.ingress.tls }}s{{ end }}://{{ $host.host }}{{ .path }}
  {{- end }}
{{- end }}
{{- else if contains "NodePort" .Values.service.type }}
Get the NodePort:

  export NODE_PORT=$(kubectl get -n {{ .Release.Namespace }} -o jsonpath="{.spec.ports[0].nodePort}" services {{ include "ranvier.fullname" . }})
  export NODE_IP=$(kubectl get nodes -o jsonpath="{.items[0].status.addresses[0].address}")
  echo "Ranvier API: http://${NODE_IP}:${NODE_PORT}"

{{- else if contains "LoadBalancer" .Values.service.type }}
Get the LoadBalancer IP (may take a few minutes):

  export SERVICE_IP=$(kubectl get svc -n {{ .Release.Namespace }} {{ include "ranvier.fullname" . }} -o jsonpath='{.status.loadBalancer.ingress[0].ip}')
  echo "Ranvier API: http://${SERVICE_IP}:{{ .Values.service.port }}"

{{- else }}
Port-forward to access Ranvier locally:

  kubectl port-forward svc/{{ include "ranvier.fullname" . }} {{ .Values.service.port }}:{{ .Values.service.port }} -n {{ .Release.Namespace }}

Then access Ranvier at: http://localhost:{{ .Values.service.port }}
{{- end }}

--------------------------------------------------------------------------------
 API ENDPOINTS
--------------------------------------------------------------------------------

Once accessible, Ranvier exposes:

  POST /v1/chat/completions    - OpenAI-compatible chat completion
  GET  /metrics                - Prometheus metrics (port {{ .Values.config.metricsPort }})
  POST /admin/backends         - Register GPU backends (requires API key)
  GET  /admin/backends         - List registered backends
  DELETE /admin/backends       - Remove a backend

Example chat completion request:

  curl -X POST http://localhost:{{ .Values.service.port }}/v1/chat/completions \
    -H "Content-Type: application/json" \
    -d '{
      "model": "llama-3",
      "messages": [{"role": "user", "content": "Hello!"}]
    }'

{{- if or .Values.auth.apiKeys .Values.auth.existingSecret }}

--------------------------------------------------------------------------------
 AUTHENTICATION
--------------------------------------------------------------------------------

Admin endpoints require authentication. Use the Authorization header:

  curl -X POST http://localhost:{{ .Values.service.port }}/admin/backends \
    -H "Authorization: Bearer <your-api-key>" \
    -d "id=1&ip=192.168.1.100&port=8080"

{{- end }}

--------------------------------------------------------------------------------
 REGISTERING BACKENDS
--------------------------------------------------------------------------------

To register GPU backends manually:

  # Register a vLLM backend
  curl -X POST "http://localhost:{{ .Values.service.port }}/admin/backends?id=1&ip=<vllm-ip>&port=8000" \
    -H "Authorization: Bearer <your-api-key>"

Consider enabling Kubernetes service discovery for automatic registration:

  backends:
    discovery:
      enabled: true
      serviceName: vllm-backends
      namespace: gpu-workloads

{{- end }}
